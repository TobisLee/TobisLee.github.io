Non-linear hypotheses

如果用逻辑回归的方法来做非线性分类，那么所需的特征量一旦很多的话，计算量会非常大。

假设我们希望训练一个模型来识别视觉对象（例如识别一张图片上是否是一辆汽车），我们怎样才能这么做呢？一种方法是我们利用很多汽车的图片和很多非汽车的图片，然后利用这些图片上一个个像素的值（饱和度或亮度）来作为特征。

假如我们只选用灰度图片，每个像素则只有一个值（而非 RGB值），我们可以选取图片上的两个不同位置上的两个像素，然后训练一个逻辑回归算法利用这两个像素的值来判断图片上是否是汽车：

我们可以看到，一个50×50的图片（非彩色），就会有2500个特征值，计算量非常大。

Gradient checking

梯度的数值检验：

ddΘJ(Θ)≈J(Θ+ϵ)−J(Θ−ϵ)2ϵ（通常ϵ=10−4）\dfrac{d}{d\Theta}J(\Theta) \approx \dfrac{J(\Theta + \epsilon) - J(\Theta - \epsilon)}{2\epsilon} \quad （通常\epsilon = 10^{-4}） dΘd​J(Θ)≈2ϵJ(Θ+ϵ)−J(Θ−ϵ)​（通常ϵ=10−4）

octave 实现：

for i = 1 : n,
	thetaPlus = theta;
	thetaPlus(i) = thetaPlus(i) + EPSILON;
	thetaMinus = theta;
	thetaMinus(i) = thetaMinus(i) - EPSILON;
	gradApprox(i) = (J(thetaPlus) - J(thetaMinus)) / (2 * EPSILON);
end;

之后，我们将 gradApprox 与 Dvec 比较，看两者值是否接近（不一定相等，约等于即可）。

Random initialzation

要想使用GD或高级优化，我们需要对 Θ\ThetaΘ 进行初始化，那么我们能否直接将它全部初始化为0呢？

不能。如果我们令所有的初始参数都为0，这将意味着我们第二层的所有激活单元都会有相同的值。也不能初始化成相同的非0常数，也会导致这种结果，这就是所谓的对称性(Symmetry)。

我们要将每一个 Θij(l)\Theta_{ij}^{(l)}Θij(l)​ 初始化成 [−ϵ,ϵ][-\epsilon, \epsilon][−ϵ,ϵ] 之间的随机数值。

octave implementation:

% init Theta2 to a random vector
Theta2 = rand(1,11) * (2 * INIT_EPSILON) - INIT_EPSILON;

One effective strategy for choosing ϵinit\epsilon_{init}ϵinit​ is to base ite on the number of units in the network.
A good choice of ϵinit\epsilon_{init}ϵinit​ is 6Lin+Lout\dfrac{\sqrt{6}}{\sqrt{L_{in} + L_{out}}}Lin​+Lout​​6​​
where Lin=slL_{in} = s_lLin​=sl​ and Lout=sl+1L_{out} = s_l + 1Lout​=sl​+1
are the number of units in the layers adjacent to Θ(l)\Theta^{(l)}Θ(l).
